# The format of the data as follows. 
# <start>; <end>; <form>; <value>
#
# <start> and <end> designate a range of codepoints. In the case of a single character
# <start> and <end> are identical.
#
# <form> is a normalization form: NFD, NFC, NFKD, or NFKC.
#
# <value> is "N", "Y", or "M".
# In testing whether text is in the specified normalization form, 
# the values have the following meanings:
# - N (NO) indicates the codepoint cannot occur
# - Y (NO) indicates that the codepoint can occur, subject to canonical ordering,
#   but without any other constraints.
# - M (MAYBE) indicates that the codepoint can occur, subject to canonical ordering,
#   but with constraints. That is, the text may not be in the specified
#   normalization form if this codepoint is preceeded by certain other characters.
#
# Other than the codepoints explicitly listed here, 
# - all assigned codepoints have the value Y,
# - all unassigned codepoints have the value M
# - all non-character codepoints (D800..DFFF, *FFFE, *FFFF) have the value N.
# - private use characters are Y, N, or M depending on the private
#   interpretation of those characters.
#
# Note: the property for unassigned characters represents the possibility that
# the software incorporating this quick check may be dealing with characters
# that originate in software using later versions of the Unicode Standard where
# those characters are assigned. The property is MAYBE since
# the canonical combining class cannot be predicted in advance.
#
# Comments are indicated with "#". Before each main section, a comment 
# shows the number of characters with that property.
#
# The comments on character ranges provides some extra information about
# the characters to make the file more readable.
# - the major category (derived from the general category, see

# - the first and last character (in UTF-8)
# - the count of numbers in the range (e.g. "=3")
# - the first and last character names.
#   - The name range is abbreviated: e.g. if the first name is "X Y Z W",
#   - and the second is "X Y A B", then "X Y Z W .. A B" is displayed
